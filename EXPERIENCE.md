# Experience with Research Intelligence OS

## Purpose
Reusable runtime, scaffolding, pipeline grammar, evaluation for research systems.

## How Builders Experience It

Use as foundation:
- runtime/pipeline-contract.md as spec.
- templates/ to bootstrap new domain agents/skills.
- evals/ for quality gates.
- scripts/ for automation.

Developers clone this, extend for a new vertical (e.g., add sociology pack).

## Agent Exploration

Agents read contracts first.
Then use templates/research-agent.agent.md as base.
Follow evaluation-contract for self-assessment.

## Usefulness

- One runtime, many domains.
- Consistent contracts = interoperable packs.
- Evaluation built-in = trustworthy agent output.
- Accelerates new research intelligence systems.
